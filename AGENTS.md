# 数学建模论文写作与文件管理规范

> 本文件约束 AI 在本目录下的一切 LaTeX 操作。

---

## 一、论文格式规范（以组委会2025修订稿为准）

详见 `format.txt`，以下为执行要点。

### 纸质版
1. A4白纸，页边距≥2.5cm，左侧装订
2. 第一页：承诺书；第二页：编号专用页（电子版不含这两页）
3. 第三页起：*摘要专用页*（含标题+关键词，不超一页，不译英文），从此编页码（阿拉伯数字，从1起，页脚居中）
4. 摘要后：正文（*不要目录**，尽量≤20页）→附录（页数不限，必须装订在一起）
5. 附录须含：支撑材料文件列表 + 全部可运行源代码。无程序则注明"本论文没有用到程序"
6. **全文不得出现参赛者身份、学校、赛区信息**

### 电子版
- 论文：单个PDF（≤20MB），内容格式与纸质版一致，第一页为摘要页
- 支撑材料：WinRAR压缩为RAR/ZIP（≤20MB），含所有源代码+数据+中间结果
- 支撑材料文件列表放入附录；无支撑材料则注明"本论文没有支撑材料"
- 承诺书和编号页不放入电子版论文和支撑材料

### 提交前检查
- [ ] 电子版PDF第一页是摘要页（无承诺书/编号页）
- [ ] 附录含全部可运行源代码
- [ ] 全文无参赛者/学校信息
- [ ] PDF≤20MB
- [ ] 支撑材料RAR/ZIP≤20MB

## 二、写作规范

### 摘要（最重要，评委优先看）
结构：背景(1-2句) → 针对问题一[方法+**具体数值结果**] → 针对问题二… → 关键词(3-5个，分号分隔)

**必须有数字，禁止空话。**

✅ "最终确定Ⅱ层厚度最小为15.7mm，皮肤外侧温度为44.05℃。"
❌ "本文取得了良好的效果。"（太空洞）

### 各节要点

- **问题重述**：1.1背景(2-3句) + 1.2问题列表(用自己的话)
- **问题分析**：每题一小节，只说方向不推导
- **模型假设**：\begin{enumerate} 编号，每条说明简化目的
- **符号说明**：\booktabs 三线表，符号列用 $...$
- **模型建立**：公式必须编号(章节-序号)，每个公式后解释变量；图表必须在正文中引用
- **模型评价**：灵敏度分析 + 客观优缺点 + 改进方向

## 三、图表与文献

### 图片
- 编号 图X.Y（X=节号），标题在下方居中
- 正文必须引用：如图~\ref{fig:xxx}~所示
- 宽度 \includegraphics[width=0.85\textwidth]{文件名}
- 存放 `figures/qN/`，格式优选PNG≥300dpi

### 表格
- 编号 表X.Y，标题在上方
- 三线表（\toprule\midrule\bottomrule），无竖线
- 正文必须引用

### 参考文献
- 正文用 \textsuperscript{[n]} 标注
- 类型标识：[R]报告 [M]专著 [J]期刊 [S]标准 [Z]其他
- 中文：作者. 标题[R]. 地点: 机构, 年份.
- 英文：Author A B. Title[J]. Journal, Year, Vol: Pages.

## 四、编译（AI 全自动，用户无需操作）

### 编译工具
- **Tectonic 路径**：C:\Users\QI\.codex\plugins\cache\openai-bundled\latex\0.2.4\bin\tectonic.exe
- **编译命令**：& "C:\Users\QI\.codex\plugins\cache\openai-bundled\latex\0.2.4\bin\tectonic.exe" -X compile --print main.tex --outdir output
- **输出位置**：output/main.pdf
- **备用方案**：如果上述路径失效，使用 Get-Command tectonic -ErrorAction SilentlyContinue 查找系统中的 tectonic

### 编译流程
每次修改 tex 后 AI 自动执行：
1. 修改 section 文件
2. 同步更新相关公共文件（见第六节）
3. 运行编译命令（见上方）
4. 检查日志，有错则修复后重编译
5. 确认 output/main.pdf 已更新

### 编译注意事项
- **字体警告**：Fontconfig error 和访问系统字体的警告可忽略
- **Overfull \hbox**：表示某行文字超出边界，可调整文本或缩小图片
- **未定义引用**：通常编译两次自动解决（Tectonic 会自动处理）
- **编译时间**：首次编译可能需要下载宏包，后续编译较快

## 六、修改问题时的联动更新规则

每次新增或修改 qN.tex 时，AI **必须同步检查并更新**：

| 文件 | 操作 |
|------|------|
| sections/symbols.tex | 检查新模型符号，**更新**符号表（增删改，保持与当前所有问题一致） |
| sections/refs.tex | 检查新引用，**更新**文献列表（增删改） |
| sections/assumption.tex | 检查新假设，**更新**假设列表（增删改） |
| sections/analysis.tex | 补充/修改对应问题的分析小节 |
| sections/appendix.tex | 更新代码清单 |

全部问题写完后用户明确要求开始写再开始：
- 填写 main.tex 中的摘要
- 撰写 sections/evaluate.tex（综合评价）

**原则是"更新"而非"一味追加"——保持所有公共文件与当前各问题内容严格一致。**

## 七、LaTeX 代码风格

- section 文件以 \section{} 开头，不含 \begin{document}
- 公式编号格式 (\thesection-\arabic{equation}})
- 行内 $...$，行间 \begin{equation}，多行 \begin{align}
- 向量 \mathbf{x}，集合 \mathbb{R}，常量 \mathrm{e}
- 图表标签用有意义英文：\label{fig:fit-comparison}
- 同一概念全文末尾一致

## 八、国赛水准的表达与排版基准（2019--2020 参考论文归纳）

### 表达
- 采用客观、简洁的学术语气；围绕“问题--方法--结果--结论”写作，少用“显然”“很好”“较为合理”等无证据判断。
- 每一段只承担一个任务：先说明目的，再给方法或公式，随后给数值结果，最后解释结果对该问题的意义。
- 每题按“模型建立 → 求解/算法 → 结果分析 → 验证或敏感性分析 → 本题小结”形成闭环；小标题应写清对象和方法，避免“分析如下”等空泛标题。
- 所有关键结论必须给可复核的数值、单位、比较对象或误差指标；首次出现的符号、参数、阈值与算法均说明来源或取值依据。
- 结论由图、表、计算或推导支撑；不以大段背景、原题复述、代码输出替代分析。模型局限性应与适用条件、改进方向对应。

### 图表与版面
- 图表少而有效：一图一结论，标题说明对象与情境；坐标轴、单位、图例、数据来源完整，正文紧邻引用并解读主要趋势。
- 表格优先呈现输入参数、方案比较和关键结果；控制列数与小数位，避免把原始数据整页堆入正文。
- 公式前交代建模含义，公式后解释新增变量和约束作用；连续推导只保留决定模型成立与求解的关键步骤。
- 正文版面保持紧凑、层级清楚：标题、正文、图表之间留适量空白；避免孤立短段、跨页大空白、超宽表格和过小图片。
- 结果图、流程图和三线表优先使用矢量或高清原图；不使用代码截图、低清网络图片或未经正文解释的装饰性图片。
